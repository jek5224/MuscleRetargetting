# Research Journal — 2026-03-31

## Stream Smooth X-Align Fix

Problem: Gastrocnemius origin contour had axes rotated ~80° from adjacent contours after stream smooth. Root cause was the x-alignment algorithm using projected-axis dot product to compare orientations across tilted planes — projection loses quadrant information when planes differ significantly.

Fix: Replaced entire stream x-smooth with simple chain propagation:
1. Level 0: pick 4-rotation (0/90/180/270°) closest to (1,0,0) using full 3D frame comparison (trace of relative rotation matrix → minimum rotation angle)
2. Level 1→end: each level picks 4-rotation closest to previous level's frame

Added `_best_4rotation()` static method that compares full 3D orientation (all 3 basis vectors) instead of a single projected axis. BP smooth handles square-like interpolation afterward.

## Pipeline Resume After Manual Cut

Bug: Two of three manual cut completion sites in `_render_manual_cut_windows` had no pipeline resume logic. Stream smooth and all subsequent steps were silently skipped for muscles completing manual cut through those paths.

Fix: Extracted pipeline resume into shared `_resume_pipeline_after_cut()` function, called from all three completion sites. Also fixed the one existing site that was calling raw `smoothen_contours_z/x/bp()` instead of `stream_smoothen_all(defer=...)`.

## Tetrahedralization Improvements

### Pole of Inaccessibility Centers
Problem: Centroid-based tet centers can fall outside non-convex contours, and centers at contour levels are coplanar with surface face vertices → degenerate tets.

Fix: Use shapely `polylabel` (pole of inaccessibility) for guaranteed-interior center points. Project contour to 2D using bounding plane basis, find pole, reconstruct 3D point via `mean + px*basis_x + py*basis_y`.

### Mid-Centers Between Levels
Added midpoints between adjacent level centers for denser tet coverage. Each surface face picks from both level centers and mid-centers, choosing largest volume.

### Cap Anchor Fix
Cap anchors (for closing mesh openings at origin/insertion) also use pole of inaccessibility instead of mean point.

### Small Boundary Loop Filtering
Problem: 4-vertex boundary loop from vertex merge artifact at insertion created degenerate cap triangles → thin tets (vol ~7e-11).

Fix: Skip boundary loops with <10 vertices in both gap closer (`_close_contour_mesh_gaps`) and tet capping. Real openings have ~30+ vertices matching the contour; tiny loops are merge artifacts.

### Results
- Gastrocnemius (2 streams, 17 levels): 0 degenerate tets, min vol 6.11e-10 (was 7.84e-11)
- Single-stream muscles: consistently 0 degenerate tets, <0.3% below 1e-8
- Biceps femoris (2 streams, 17+8 levels): 0 degenerate, 8.6% below 1e-8 (cut boundary thin triangles)
