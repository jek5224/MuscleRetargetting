# Research Journal — 2026-03-06

## Root Cause: DOF Range Mismatch

### Discovery
- NN oscillation root cause found: training ROM was anatomical literature values (e.g. hip flexion [-30°, 120°])
- But DART's BallJoint uses exponential map — BVH values span ±180° for hip rotation
- Checked all 82 LaFAN1 BVHs: 6-7 out of 7 DOFs exceeded training ROM for every file
- Network was extrapolating far outside training domain

### Fix: Widened DOF Ranges
- Computed actual BVH ranges + 10% margin
- Hip x/y: ±30-45° → ±195-216° (exponential map wraps)
- Ankle z: [-20°, 50°] → [-70°, 155°]
- All 7 DOFs now cover observed BVH data

### GPU Inference Fix
- Viewer was hardcoded to `device='cpu'` — ~50-100ms/frame
- Changed to CUDA + cached PCA tensors on GPU — ~3ms/frame

### Constraint Losses Added
- Fixed vertex loss (λ=10): penalizes displacement at origin/insertion vertices
- Inter-muscle loss (λ=1): penalizes distance deviation between adjacent muscle vertices
- Fixed preprocessing: was only saving 2-5 anchor points, now saves all fixed vertices (cap faces + contour matches, ~50-80 per muscle)
- Fixed PCA reconstruction bug: constraint loss was missing `+ pca_means`

### 30K Rebake on A6000
- Added `--end` parameter for splitting samples across GPUs
- 4 parallel sbatch jobs: 2 regions × 2 halves (15K each) on 4 GPUs
- Speed: ~0.58s/sample (L_UpLeg), ~0.76s/sample (L_LowLeg)
- ETA: ~3 hours total
